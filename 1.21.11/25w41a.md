pack breaking | Data pack format has been increased to `89.0`.

pack breaking | Resource pack format has been increased to `70.0`.

network | Minecraft Server Management Protocol version has been increased to `1.1.0`.

network | Added `server/activity` notification to Server Management Protocol, which indicates that a network connection has been initiated. Rate limited to one notification per 30 seconds.

jar | Added `ClientFps` JFR event.

network | Experience packets that do not change the player's experience progress or level are no longer sent by the server.

command | Added `stopwatch` command for creating stopwatches that track real time. Stopwatches do not depend on game ticks and only run when the server/world is running.

command | Added `execute if|unless stopwatch <id> <range>` syntax to compare the elapsed time of a stopwatch with a floating point value range. The stopwatch value is measured in seconds with millisecond accuracy.

component | Added `use_effects` component to control player behavior when using an item. Fields: `can_sprint` (boolean, defaults to `false`) and `speed_multiplier` (float `0.0` to `1.0`, defaults to `0.2`).

component | Added `minimum_attack_charge` component to set the minimum attack charge on the attack indicator required to attack with an item. Format: float from `0.0` (no charge required) to `1.0` (full charge required).

component | Added `damage_type` component to specify the type of damage an item deals. Format: string damage type identifier, e.g. `damage_type='minecraft:spear'`.

component | Added `kinetic_weapon` component to enable charge-type attacks where damage is dealt along a ray every tick based on relative speed of entities. Fields: `min_reach`, `max_reach`, `hitbox_margin`, `delay_ticks`, `dismount_conditions`, `knockback_conditions`, `damage_conditions`, `forward_movement`, `damage_multiplier`, `sound`, and `hit_sound`.

component | Added `piercing_weapon` component to enable quick attacks that damage multiple entities along a ray. Fields: `min_reach`, `max_reach`, `hitbox_margin`, `deals_knockback`, `dismounts`, `sound`, and `hit_sound`.

component | Added `swing_animation` component to specify the swing animation when attacking or interacting. Fields: `type` (string: `none`, `whack`, or `stab`, defaults to `whack`) and `duration` (integer, defaults to `6`).

component breaking | Changed `consumable` component. The `animation` field value `spear` has been renamed to `trident`, and a new `spear` animation has been added.

component | Items with `intangible_projectile` component now show information about it in their tooltip.

entity nbt breaking | The `AngryAt` field has been renamed to `angry_at`.

entity nbt breaking | The `AngerTime` field has been removed and replaced with `anger_end_time`, containing the time anger ends in game ticks.

damage-type | Added `spear` damage type.

predicate | Component predicates now include predicates for checking existence of every component type using the format `{<component_type>: {}}`. The predicate passes as long as the component is present, regardless of value.

predicate | The `flags` predicate in entity predicates now supports `is_in_water` (when an entity is touching water or a bubble column) and `is_fall_flying` (when an entity is gliding with an elytra).

predicate | Item predicates in command form now accept empty predicates for any component type to check for component existence. The syntax `*[instrument]` and `*[instrument~{}]` are equivalent.

enchantment | Added `post_piercing_attack` enchantment effect component for effects applying after a piercing attack with an item.

enchantment | Added `apply_impulse` enchantment entity effect to apply an impulse to the targeted entity. Fields: `direction`, `coordinate_scale` and `magnitude`.

enchantment | The `play_sound` enchantment entity effect can now support a list of sound event identifiers, one for each enchantment level. If a level exceeds the number of sounds, the last sound is used.

enchantment | Added `exponent` that raises the base to the specified power level. Fields: `base` and `power`.
