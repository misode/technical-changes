pack breaking | Resource pack format has been increased to `79.0`.

text | Tags resolved from `minecraft:nbt` text components with `interpret: false` are now pretty-printed instead of being flattened into a single text component.

recipe | The `result` field now accepts short form across all recipe types (e.g. `"minecraft:foo"` is equivalent to `{"id":"minecraft:foo", "count": 1}`).

environment-attribute | Updated the `gameplay/turtle_egg_hatch_chance` environment attribute to have a default value of `0.002`.

component | Added `minecraft:dye` data component, which represents a color of dye. Used in various places that previously required the exact dye item, including mob and block interactions.

loot | Changed `minecraft:set_random_dyes` loot function. It no longer works only for items in the `#dyeable` tag.

recipe breaking | The `minecraft:crafting_special_mapcloning` recipe type has been removed. Functionality has been absorbed by `minecraft:crafting_transmute`.

recipe | Added `show_notification` field to some recipe types. This optional boolean describes if a popup should be shown when the recipe is unlocked, and defaults to `true`.

recipe breaking | Removed unused `group` field from recipes without a recipe book: `minecraft:stonecutting`, `minecraft:smithing_transform`, and `minecraft:smithing_trim`.

recipe | Changed `minecraft:crafting_transmute` recipe type. Recipes can now accept multiple items matched by the `material` ingredient.

recipe breaking | Changed `minecraft:crafting_dye` recipe type, which replaces `minecraft:crafting_special_armordye`.

recipe breaking | Added `minecraft:crafting_imbue` recipe type, which replaces `minecraft:crafting_special_tippedarrow`. This recipe copies the `minecraft:potion_contents` component from the source item to the result.

recipe breaking | Changed `minecraft:crafting_special_bannerduplicate` recipe type. The recipe still only works for banner items.

recipe breaking | Changed `minecraft:crafting_special_bookcloning` recipe type. The number of items matched by the material ingredient beyond the first one will be added to the resulting stack size.

recipe breaking | Changed `minecraft:crafting_decorated_pot` recipe type. Items matched by ingredients will be added to the matching fields in the `minecraft:pot_decorations` component.

recipe breaking | Changed `minecraft:crafting_special_firework_rocket` recipe type. Exactly one stack matching the shell ingredient must be present. The number of stacks matching the fuel ingredient controls the `flight_duration` field. The `explosions` field is copied from the `minecraft:firework_explosion` component of the star ingredient if any.

recipe breaking | Changed `minecraft:crafting_special_firework_star_fade` recipe type. The recipe now uses the `minecraft:dye` component. The `fade_colors` field of `minecraft:firework_explosion` is set to the values from the dye ingredients.

recipe breaking | Changed `minecraft:crafting_special_firework_star` recipe type. The recipe now uses the `minecraft:dye` component.

recipe breaking | Changed `minecraft:crafting_special_mapextending` recipe type. The source item is kept in the crafting grid.

recipe breaking | Changed `minecraft:crafting_special_shielddecoration` recipe type. The result is built by transmuting the target item and copying the `minecraft:banner_patterns` component from the banner item and setting the `minecraft:base_color` component based on the banner type.
