component | The `piercing_weapon` and `kinetic_weapon` components now have bounds on their reach parameters: `min_reach` (`0.0` to `128.0`), `max_reach` (`0.0` to `128.0`), and `hitbox_margin` (`0.0` to `1.0`).

data | Added timelines system, defined in data packs under the `timeline` registry. Timelines specify game behavior based on absolute day time and can optionally repeat over a period. Format: object with optional `period_ticks` and `tracks` fields.

data | Added `blend_to_gray` RGB/ARGB modifier that modifies a color by taking its grayscale form, applying a brightness modifier, and mixing with a factor. Format: object with `brightness` (float `0.0` to `1.0`) and `factor` (float `0.0` to `1.0`) fields.

Format: string ID (`full_moon`, `waning_gibbous`, `third_quarter`, `waning_crescent`, `new_moon`, `waxing_crescent`, `first_quarter`, or `waxing_gibbous`), defaults to `full_moon`. Only visible with overworld skybox.

environment-attribute | Added `visual/star_brightness` Environment Attribute to control brightness of stars in the sky, where `0.5` is normal brightness during night and `0` is fully hidden. Format: float between `0.0` and `1.0`, defaults to `0.0`. Only visible with overworld skybox.

environment-attribute | Added `visual/sky_light_factor` Environment Attribute to control visual brightness of sky light. Format: float, defaults to `1.0`. Value passed to `lightmap.fsh` shader as `SkyFactor`.

environment-attribute | Added `audio/firefly_bush_sounds` Environment Attribute to control whether firefly bushes produce idle sounds when not below an opaque block. Format: boolean, defaults to `false`.

environment-attribute | Added `gameplay/sky_light_level` Environment Attribute to control effective light level of the sky used by mechanics such as mob spawning or daylight detectors. Format: float, defaults to `15.0`. Can only be specified on dimensions.

environment-attribute | Added `gameplay/eyeblossom_open` Environment Attribute to control whether eyeblossoms open or close. Format: `true`, `false`, or `"default"`, defaults to `"default"`.

environment-attribute | Added `gameplay/turtle_egg_hatch_chance` Environment Attribute to control chance that a turtle egg will switch to its next hatching state when randomly ticked. Format: float between `0.0` and `1.0` defaults to `0.0`.

environment-attribute | Added `gameplay/creaking_active` Environment Attribute to control whether creaking heart blocks and their corresponding creaking are active. Format: boolean, defaults to `false`.

environment-attribute | Added `gameplay/surface_slime_spawn_chance` Environment Attribute for additional chance rolled when a natural slime spawn attempt occurs in a biome with the `#allows_surface_slime_spawns` tag. Format: float between `0.0` and `1.0`, defaults to `0.0`.

environment-attribute | Added `gameplay/cat_waking_up_gift_chance` Environment Attribute to control chance that a cat lying on its owner's bed will drop a gift when the player wakes up. Format: float, defaults to `0.0`.

environment-attribute | Added `gameplay/monsters_burn` Environment Attribute to control whether monsters burn while exposed to the sky. Format: boolean, defaults to `false`.

environment-attribute | Added `gameplay/can_pillager_patrol_spawn` Environment Attribute to control whether pillager patrols can spawn. Format: boolean, defaults to `true`. Replaces `#without_patrol_spawns` biome tag.

environment-attribute | Added `gameplay/baby_villager_activity` Environment Attribute to control default AI activity for baby villagers. Format: mob activity, defaults to `idle`. Supported activities: `core`, `hide`, `idle`, `meet`, `panic`, `play`, `pre_raid`, `raid`, and `rest`.
